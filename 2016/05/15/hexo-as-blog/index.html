How to use Hexo as your blog ? (Part 1)

May 15, 2016

Recently I wanted to change my Github page to a blog. I was looking for some automated options, where I don’t have to build yet another blog. But I wanted it to be simple and minimalistic.

Github do provide an option of Jekyll. It looks really cool but I am not much familiar with Ruby language yet. I wanted something that I could tweak easily.

So I started looking for something in Node realm. Then I found Hexo, its simple to setup and use. Its like Jekyll in JavaScript.

Let see how you can setup your first blog.

Installing and Creating

Hexo is available as a CLI by using npm. Lets install this package for global use.

Once Hexo CLI has been installed, command hexo will be available. Now its time to create a new blog. Its simple as this command.

Hexo will create a folder called my-blaggg. New version of Hexo does npm install itself. Basically your blog is now ready for creating new posts.

Configure

Before starting we should at least change name of our blog. This can be done by modifying a file called _config.yml, you can find it under my-blaag folder.

It offers various option to configure but we will start with basics. We need to change the name and timezone for now.

Timezone can be any valid moment.js timezone string. Whole list of timezone is available here.

Installing a theme

Hexo supports plugins and themes. I myself didn’t like the stock theme very much. Here are various themes that Hexo community provides.

After you have selected any theme you have to clone it under your themes folder. Under your blog directory you can run this command.

This will clone my blog’s current theme to your local blog. Now you have to change config settings to use this new theme.

Under your blog’s (not your theme’s config) _config.yml you have to change the theme parameter to apollo.
